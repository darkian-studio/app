# Contributing

Thank you for wanting to help improve Darkian Studio!

**This repository does not contain the application source code.** Darkian Studio
is proprietary, and its code lives in a private repository. As a result, this
repo accepts **no code pull requests**.

What you *can* contribute here:

- **Bug reports** using the Bug report template.
- **Feature requests** using the Feature request template.
- **Discussions** (questions, ideas, show-and-tell) in GitHub Discussions.
- **Improvements to this support repo's content** — fixes to the README,
  issue/feature templates, `install.sh`, or documentation — are welcome as
  pull requests.

## Reporting a bug well

1. Read the install steps in the README.
2. Run the setup command and tap **Verify setup** in DS.
3. Gather **Diagnostics logs** (Settings → Diagnostics → Diagnostics logs →
   copy) and any **Crash report**.
4. Open a Bug report and paste them in. Keep timestamps and error text.

## Behavior expectations

- Be respectful and constructive.
- Search existing issues/discussions before posting.
- Do not post private/secret material (tokens, keys, personal data).

Maintainers may close issues that lack the minimum reproduction information or
fall outside the scope of this support repo.
